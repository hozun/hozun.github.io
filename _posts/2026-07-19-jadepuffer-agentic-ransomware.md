결론부터. 이번 달 보안 업계에서 가장 주목할 사건은 새로운 취약점이 아니다. **공격의 오케스트레이션 주체가 사람에서 AI 에이전트로 넘어간 첫 문서화된 사례**, JadePuffer다. 7월 초 Sysdig 위협 연구팀이 공개했다.

개별 기법은 전부 몇 년 된 것들이다. 새로운 건 이걸 **누가 엮었느냐**다. 정찰 → 크리덴셜 탈취 → 측면 이동 → 지속성 확보 → 권한 상승 → DB 파괴까지, 사람 오퍼레이터가 단계마다 판단하던 일을 LLM 에이전트가 스스로 추론하며 실시간으로 처리했다.

이 글을 읽으면 좋은 이유: 우리가 [AI 에이전트](/2026/03/27/agentic-ai-multi-agent-guide/)를 생산성 도구로 논할 때, 같은 구조가 공격 측에서 어떻게 무기화되는지를 구체적으로 보여주기 때문이다.

---

## 1. 무슨 일이 있었나 — 한 문단 요약

인터넷에 노출된 **Langflow**(오픈소스 LLM 앱 프레임워크) 인스턴스가 시작점이었다. 공격자는 `CVE-2025-3248`(Langflow 검증 엔드포인트의 인증 없는 Python 코드 실행)로 초기 침투한 뒤, 그 안에서 **AI 에이전트를 돌려** 나머지를 자동화했다. 에이전트는 클라우드 크리덴셜을 쓸어 담고, 다른 서버로 옮겨가고, 백도어 관리자 계정을 심고, 최종적으로 프로덕션 데이터베이스를 암호화·파괴한 뒤 협박 문구를 남겼다.

Base64로 인코딩된 Python 페이로드가 계속 날아왔는데, 그 코드 안에 **자연어 주석**이 잔뜩 들어 있었다. 사람이 짠 공격 스크립트에는 좀처럼 없는 패턴 — LLM이 자기 의도를 서술하며 코드를 생성한 흔적이다.

---

## 2. 공격 체인 — 단계별로

```
[초기 침투]  Langflow (CVE-2025-3248, 인증 없는 RCE)
     │
     ▼
[1단계: 침투 호스트에서]
  · 시스템/프로세스/네트워크 인터페이스 정찰
  · 크리덴셜 스윕
      - LLM API 키 (OpenAI / Anthropic / DeepSeek)
      - 클라우드 키 (Alibaba/Aliyun/Tencent/Huawei 등 중국계 명시적 커버)
      - 암호화폐 지갑, DB 크리덴셜
  · MinIO 오브젝트 스토어 → 기본 계정(minioadmin:minioadmin)으로
      terraform-state 버킷에서 credentials.json / .env 추출
  · crontab 비콘 설치 (30분마다 45.131.66[.]106:4444)
     │
     ▼
[2단계: 프로덕션 DB로 피벗]
  · Nacos 인증 우회 (CVE-2021-29441 + 기본 JWT 키)
  · MySQL root 접근 → 백도어 관리자 계정 주입
  · 컨테이너 이스케이프 정찰 (docker.sock, /proc/1/cgroup 탐지)
  · AES_ENCRYPT()로 DB 암호화 (키는 메모리에만, 저장 안 함)
  · 스키마 전체 파괴 + "스테이징 서버로 유출 완료" 협박
```

주목할 점: **모든 취약점(CVE-2025-3248, CVE-2021-29441)과 기법(기본 크리덴셜, JWT 기본 키)이 이미 알려진 것들**이다. 방어 관점에서 이건 나쁜 소식이자 좋은 소식이다. 나쁜 소식은 진입 장벽이 사라졌다는 것, 좋은 소식은 **기본만 지켰어도 막혔다**는 것.

---

## 3. "사람이 아니라 에이전트였다"는 증거

Sysdig가 이 사건을 '에이전틱'이라고 부르는 근거는 네 가지다. 이 부분이 이 사례의 진짜 알맹이다.

### (1) 자기 서술형 페이로드

페이로드 코드에 이런 자연어 주석이 박혀 있었다.

> `"High-ROI databases to drop (data already backed up to...)"`
> `"REDACTED is largest - drop it too"`

타깃을 고르는 이유를 코드 안에서 스스로 설명한다. 사람 오퍼레이터는 이렇게 안 짠다. LLM이 "생각하면서 코드를 뱉는" 반사적 패턴이다.

### (2) 31초 만의 적응형 수정

Nacos 백도어 삽입이 실패했다. 원인은 subprocess의 `PATH` 문제로 bcrypt 해시가 빈 값이 된 것. 에이전트는—

- **12초** 안에 근본 원인을 진단하고
- **19초** 뒤에 수정 페이로드를 발사

수정 내용도 정교하다. 잘못 만들어진 계정을 지우고 → `subprocess.run()` 대신 `import bcrypt` 직접 호출로 전환 → 새 해시 생성 → 재삽입. 이 다단계 추론을 31초에 끝냈다. 사람 오퍼레이터의 반응 속도를 넘어선다.

### (3) 실패 맥락을 파싱한 외과적 교정

`DROP DATABASE`가 외래 키 제약 때문에 조용히 실패하자, 다음 페이로드는 명령을 `SET GLOBAL FOREIGN_KEY_CHECKS=0`으로 감싸고 실행 후 원복했다. 무작정 재시도가 아니라 **에러 컨텍스트를 읽고 정확히 그 문제만 고친** 것.

### (4) 응답 형식 변화에 대한 이해

MinIO 열거 때 `?format=json`을 요청했는데 XML이 돌아왔다. 에이전트는 원 요청을 재시도하지 않고, **바로 다음 페이로드에서 파서를 S3 XML 스키마에 맞게 바꿨다.** 패턴 매칭이 아니라 이해에 가깝다.

## 4. 그런데 완전 자율이었나 — 회색지대

과대 해석은 경계할 필요가 있다. 자동화된 부분과 사람이 설정했을 법한 부분을 나눠 보면:

| 확실히 자동화됨 | 사람이 설정했을 가능성 |
|---|---|
| 초기 정찰·크리덴셜 스윕 | 협박에 쓴 비트코인 주소 |
| MinIO 버킷 열거·객체 추출 | (아래 설명) |
| Nacos 우회·백도어 주입 | |
| 컨테이너 이스케이프 사전 점검 | |
| DB 암호화·파괴, crontab 지속성 | |

협박에 쓴 비트코인 주소(`3J98t1WpEZ73CNmQviecrnyiWrnqRhWNLy`)는 사실 **비트코인 문서에 나오는 P2SH 예제 주소**다. LLM 학습 데이터에 수없이 등장하는 값이라, 에이전트가 그냥 환각으로 뱉었을 수도 있고 오퍼레이터가 우연히 같은 주소를 통제 지갑으로 걸어둔 것일 수도 있다. 체인 분석상 737건의 확정 트랜잭션에 입금 즉시 스위핑 정황이 있어 판단이 갈린다 — **에이전트의 시스템 프롬프트를 봐야 결론이 난다.**

즉, "완전 무인 랜섬웨어"라기보단 **사람이 목표와 틀만 정하고, 실행의 대부분을 에이전트에 위임한** 형태로 보는 게 정확하다. 그래도 위임의 범위가 이전과는 차원이 다르다.

---

## 5. 그래서 뭐가 바뀌나

한 줄로: **"랜섬웨어의 스킬 하한선이 에이전트 돌리는 비용까지 내려갔다."**

예전엔 정찰·측면 이동·권한 상승·지속성을 각각 잘하는 오퍼레이터의 숙련도가 필요했다. 이제는 그 숙련도를 LLM 오케스트레이션이 대체한다. 어느 단계도 깊이 몰라도 체인 전체가 굴러간다. 이건 [에이전트가 도구를 엮어 목표를 달성하는 구조](/2026/04/18/agent-agentic-mcp-skills-evolution/)를 우리가 생산성 측면에서 반겼던 것과 **정확히 같은 메커니즘**이다. 방향만 반대일 뿐.

---

## 6. 방어 관점 체크리스트

역설적이게도 대응책은 화려하지 않다. 이번 공격이 낡은 기법의 조합이었기 때문이다.

- **패치**: `CVE-2025-3248`(Langflow), `CVE-2021-29441`(Nacos)은 여전히 광범위하게 악용된다. 인터넷 노출된 Langflow/Nacos는 1순위 표적.
- **크리덴셜 분리**: 프로바이더 API 키·클라우드 크리덴셜을 **웹에서 닿는 오케스트레이션 서비스와 같은 곳에 두지 말 것.** Langflow가 매력적인 진입점이었던 이유가 바로 환경변수에 클라우드 키가 같이 있었기 때문이다.
- **기본 계정 제거**: `minioadmin:minioadmin`, 기본 JWT 서명 키, 인터넷 노출 DB root — 전부 이번 체인의 시작점.
- **이그레스 통제**: 침해된 앱 호스트가 외부 스테이징 서버나 임의 C2로 나가지 못하게.
- **런타임 탐지의 반전된 이점**: 이번엔 페이로드에 자연어 주석이 잔뜩 있었다. **DB 프로세스 출력에 섞인 자연어 서술은 그 자체로 이상 징후 시그니처**가 된다. 에이전트가 수다스러운 건 방어자에겐 선물이다.

---

## 마치며

JadePuffer의 개별 조각은 하나도 새롭지 않다. 새로운 건 **"엮는 지능"이 상품화됐다**는 사실이다. 우리가 [멀티 에이전트](/2026/03/27/agentic-ai-multi-agent-guide/)로 업무를 자동화하는 바로 그 능력이, 공격 측에서는 침투 후 단계를 자동화한다. 도구는 중립적이고, 그래서 방어의 기본기가 다시 중요해진다.

기본 크리덴셜 하나, 안 한 패치 하나가 이제는 사람 대신 지치지 않는 에이전트에게 열려 있다고 생각하면 된다.
