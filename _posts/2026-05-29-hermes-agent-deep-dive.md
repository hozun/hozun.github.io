오늘 두 편의 글로 [하네스 엔지니어링이란 무엇인지](/2026/05/29/harness-engineering/) 그리고 [한 턴 안에서 하네스가 어떻게 동작하는지](/2026/05/29/harness-anatomy-visual/) 정리했다. 마침 같은 주에 화제가 된 **Hermes Agent**가 좋은 사례라서 이걸 하네스 관점에서 한 번 뜯어본다.

결론부터: Hermes Agent는 "**오픈소스로 풀어둔 하네스**"다. Claude Code나 Cursor가 호스팅된 폐쇄형 하네스라면, Hermes는 내가 내 서버에서 돌리고 코드도 다 까볼 수 있는 하네스다. 그리고 그 안에서 가장 신경 쓴 부분이 **메모리와 스킬 진화**다.

특이점은 **다중 채팅 플랫폼**이다. Telegram, Discord, Slack, WhatsApp, Signal, CLI — 한 게이트웨이로 묶어서 같은 에이전트를 어디서 말 걸어도 같은 컨텍스트로 응답한다.

## 하네스 관점에서 본 Hermes의 구성

[해부도 글](/2026/05/29/harness-anatomy-visual/)에서 봤던 하네스의 6개 블록(컨텍스트 조립 → 모델 호출 → 응답 파서 → 권한 게이트 → 툴 실행 → 결과 반영) 위에 Hermes를 매핑해보면 이렇게 된다.

```
[다중 입력 게이트웨이]
  Telegram / Discord / Slack / WhatsApp / Signal / CLI
        │
        ▼
[컨텍스트 조립기]
  + 사용자 프로파일 (Persistent Memory)
  + 활성 스킬 문서들 (Self-Evolving Skills)
  + 환경 정보 (현재 실행 환경: 로컬 / Docker / SSH / Modal)
        │
        ▼
[모델 라우터]  ◄── OpenRouter / Nous Portal / vLLM / 커스텀
        │
        ▼
[툴 실행기]  40+ 내장 스킬
  - 브라우저: 검색, 페이지 추출, 자동화, 비전 분석, 이미지 생성
  - 실행 환경: 로컬 터미널, Docker, SSH 원격, Modal/Singularity
  - 파일/코드/통신 등
        │
        ▼
[궤적 저장소]
  → 메모리 갱신 + 새 스킬 작성/정제
  → (선택) ShareGPT 포맷으로 내보내 RL 학습에 재사용
```

다른 에이전트와 가장 다른 두 박스는 **컨텍스트 조립기에 박힌 Persistent Memory**와 **궤적 저장소가 만드는 Self-Evolving Skills**다.

---

## 핵심 차별점 1 — Persistent Memory

대부분의 에이전트는 한 세션이 끝나면 사용자가 누군지, 뭘 좋아하는지, 어떤 프로젝트를 진행 중인지 다 잊는다. ChatGPT의 메모리 기능이나 Claude의 Projects 같은 흉내가 있지만 그건 "특정 채팅" 안에서만 산다.

Hermes의 Persistent Memory는 다르다.

- 사용자 선호도·진행 중인 프로젝트·실행 환경을 **에이전트 인스턴스 단위**로 보존한다
- 어느 채널(Slack/Discord/CLI 등)로 말을 걸어도 같은 메모리에 접근
- 메모리는 단순 텍스트 노트가 아니라 **구조화된 파일**로 디스크에 저장 — 사용자가 직접 열어 보고 편집 가능

하네스 엔지니어링 관점에서 이건 큰 결정이다. 매 호출마다 시스템 프롬프트 뒤에 자동 주입되는 "프로젝트 사실들"이 점점 풍부해지고, 사용자는 모델한테 자기소개를 반복할 필요가 없다.

---

## 핵심 차별점 2 — Self-Evolving Skills

더 흥미로운 쪽은 스킬이다. 공식 표현으로는:

> "Hermes writes and refines its own skills. Every time the agent encounters a complex task or receives feedback, it saves its learnings as a skill."

쉽게 풀면 이렇다.

1. 에이전트가 복잡한 작업을 해낸다
2. 그 과정 자체를 **재사용 가능한 스킬 문서**로 저장한다 (절차·체크리스트·자주 쓰는 명령 등)
3. 다음에 비슷한 작업이 들어오면 그 스킬을 컨텍스트에 자동 로드
4. 새로 배운 게 있으면 기존 스킬을 정제·갱신

기존 "프롬프트 엔지니어링 + RAG"와 비슷해 보이지만, 차이는 **에이전트가 자기 스킬을 직접 쓰고 고친다**는 점이다. 사람이 프롬프트를 다듬어 주입하는 게 아니라, 에이전트가 자기 경험을 스킬로 결정화하는 루프.

이게 작동하려면 두 가지가 받쳐줘야 한다:

- **메모리 정책**: 모든 경험을 다 저장하면 노이즈 폭발. 무엇을 스킬로 결정화할지 정책이 필요
- **지속 실행 환경**: 한 번 쓰고 종료되는 게 아니라 24/7 돌면서 누적해야 의미가 있다

후자 때문에 NVIDIA가 같이 띄운 게 **DGX Spark + Hermes** 조합이다. 128GB 통합 메모리로 큰 모델을 로컬에서 24/7 돌리고, Tensor Core가 스킬 정제를 초 단위에 끝내는 시연.

## 그래서 다른 에이전트와 뭐가 다른가

이미 쓸 만한 에이전트가 많은데 Hermes만의 자리는 어디인지 비교해본다.

| | Claude Code | Cursor | Hermes Agent |
|---|---|---|---|
| 호스팅 | Anthropic | Cursor | **셀프 호스팅** |
| 라이선스 | 비공개 | 비공개 | **MIT** |
| 주 인터페이스 | CLI / IDE | 에디터 | **멀티 채팅 채널** |
| 모델 선택 | 주로 Claude | Claude/GPT/기타 | **OpenRouter 200+ / vLLM 로컬** |
| 세션 간 메모리 | 제한적 | 프로젝트 룰 | **풀 Persistent Memory** |
| 학습 루프 | 없음 | 없음 | **Self-Evolving Skills** |
| 주 용도 | 코딩 | 코딩 | **범용 비서·자동화** |
| 데이터 위치 | 공급자 클라우드 | 공급자 클라우드 | **본인 서버** |

코딩 한정으로 보면 Claude Code/Cursor가 여전히 더 매끄럽다. 다만 Hermes의 자리는 **"내 데이터로, 내 서버에서, 멀티 채널로 항상 돌아가는 개인 에이전트"**라는 좀 다른 카테고리다.

---

## 트레이드오프

- **운영 부담**: 셀프 호스팅은 자유와 함께 책임이 온다. 모델 비용·VPS·키 관리 다 본인 몫
- **코딩 도구로서의 깊이**: 같은 워크플로 깊이로 보면 Claude Code/Cursor가 더 다듬어져 있다. Hermes는 "범용"에 가까운 트레이드오프
- **메모리 폭주 위험**: Persistent Memory와 Self-Evolving Skills 모두, 정책 없이 두면 컨텍스트 오염의 원천이 된다. 결국 [지난 글에서 본 컨텍스트 압축 정책](/2026/05/29/harness-anatomy-visual/)이 필요
- **학습 루프의 진정한 효용은 아직 측정 중**: "자기 개선" 주장이 실제로 시간이 지날수록 의미 있는 향상으로 누적되는지는 더 두고 봐야 할 영역

---

## 누구한테 추천할 만한가

추천:

- 데이터를 외부에 보내기 싫은 환경 (사내·연구실·민감 데이터 다루는 팀)
- 모델을 자유롭게 갈아끼우며 비교하고 싶은 사람
- 코딩이 아니라 **일상 자동화 비서**가 필요한 사람 (Telegram/Slack에 말 거는 식)
- 에이전트 하네스 내부 구조를 코드로 읽으며 배우고 싶은 사람

비추천:

- "그냥 잘 돌아가는 코딩 도구"가 필요한 사람 — Claude Code/Cursor가 더 빠르다
- VPS·키·모델 관리에 시간 쓸 여유가 없는 사람
- 메모리·스킬 정책을 직접 손볼 의향이 없는 사람 — 기본값으로 두면 시간이 지날수록 더러워진다

## 마무리

Hermes Agent는 그 자체로 좋은 도구지만, 하네스 엔지니어링을 공부하는 사람에게 더 큰 가치가 있다. **닫힌 도구의 결과만 보는 게 아니라, 잘 설계된 하네스 한 채를 통째로 읽을 수 있는 오픈소스 사례**이기 때문이다.

오늘 세 편의 글을 한 줄로 묶자면: 모델은 평준화되고, 하네스가 차별화 포인트가 되고, 그 하네스를 오픈소스로 풀어둔 시도가 시장에서 가장 빠르게 사용량을 끌어모으고 있다는 것. 다음 1년의 흐름을 미리 보고 싶다면 Hermes 코드를 한 번 까볼 만하다.
